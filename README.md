# SE_experiments_fs
- 实验目标
    了解通过pull request开展代码审查的方法
    
- 实验内容
在Github、Gitee等平台上开展基于pull request的代码审查，了解轻量级的评审方法如何应用在项目开发过程中。实验步骤和要求如下：
    - – 在远程仓库建立Git项目；
    - – 以开发者身份从Git仓库clone项目代码；
    - – 开发者修改项目代码，并通过pull request提交代码；
    - – 评审者审查代码，给出回复意见，形成评审不通过的结论；
    - – 开发者再次修改代码，重新提交代码；– 评审者评审通过；
    - – 开发者检查前次更新是否最终提交
